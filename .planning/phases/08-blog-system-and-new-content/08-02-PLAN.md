**Content structure:**

But security practices often lag behind deployment speed. Most breaches are misconfigurations, not sophisticated attacks.

2. **Common cloud misconfigurations we find** (4-5 specific items with detail):
   - Overly permissive IAM roles (admin access where read-only suffices)
   - S3/Blob storage with public access or weak access policies
   - No encryption at rest for databases and object storage
   - Missing CloudTrail/Activity Log monitoring
   - Default security groups allowing broad inbound access
     For each: what it means, why it matters, one-line fix.

3. **Data sovereignty and regulatory considerations** (2-3 paragraphs): Where data lives matters for BNR-regulated entities. AWS Africa region vs EU/US regions. Rwanda Data Protection Law implications. Hybrid cloud approaches for compliance-sensitive workloads.

AWS/Azure policies on penetration testing (what's allowed without pre-approval).

5. **Building a cloud security baseline** (bullet checklist): Enable MFA everywhere, least-privilege IAM, encrypt at rest and in transit, enable logging, review security groups quarterly, use infrastructure-as-code for drift detection.
