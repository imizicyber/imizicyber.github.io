## Spacing Scale

Declared values (multiples of 4, 8px base grid):

| Token | CSS Custom Property | Value         | Usage                                                  |
| ----- | ------------------- | ------------- | ------------------------------------------------------ |
| xs    | `--space-1`         | 4px (0.25rem) | Icon gaps, inline padding, tight element margins       |
| sm    | `--space-2`         | 8px (0.5rem)  | Compact element spacing, button icon gaps              |
| md    | `--space-4`         | 16px (1rem)   | Default element spacing, card internal padding         |
| lg    | `--space-6`         | 24px (1.5rem) | Section heading margins, card gaps in grids            |
| xl    | `--space-8`         | 32px (2rem)   | Layout gaps between card groups, form field spacing    |
| 2xl   | `--space-12`        | 48px (3rem)   | Section vertical padding (top/bottom)                  |
| 3xl   | `--space-16`        | 64px (4rem)   | Major section breaks, hero vertical padding            |
| 4xl   | `--space-20`        | 80px (5rem)   | Page-level top/bottom spacing, hero padding on desktop |

Additional tokens for intermediate use:

| Token | CSS Custom Property | Value          | Usage                          |
| ----- | ------------------- | -------------- | ------------------------------ |
| -     | `--space-3`         | 12px (0.75rem) | Badge padding, small card gaps |
| -     | `--space-10`        | 40px (2.5rem)  | Medium section breaks          |

Exceptions:

- Nav height: 56px (existing, preserved for E2E test compatibility)
- Hero top padding on desktop: 132px (nav height 56px + 76px breathing room -- existing value preserved)
- Touch targets: 44px minimum width and height on all interactive elements (accessibility requirement from CONTEXT.md)

**Source:** RESEARCH.md Architecture Patterns, Pattern 4. Current codebase uses ad-hoc spacing (132px, 48px, 28px, 24px). This scale systematizes existing values.

### Accent Reserved-For List (explicit)

The accent color `--acc` (#34d399) and its variants are ONLY used on these specific elements:

1. "Book a Consultation" CTA button (background: `--accdark`, hover: `--acc2`)
2. Section tag labels (color: `--acc`, font: JetBrains Mono)
3. Terminal cursor animation in hero (background: `--acc`)
4. Credential badge type indicators (color: `--acc`)
5. Active/current nav link indicator
6. Hero subtitle inline highlight (color: `--acc`)
7. FAQ question chevron/arrow on hover
8. Card hover border accent (1px solid with low opacity: `rgba(52, 211, 153, 0.15)`)
9. Contact form submit button (same treatment as primary CTA)
10. Selection highlight (`::selection` background)

Accent is NOT used for: body text, backgrounds of content sections, decorative fills, borders at rest, or non-interactive elements.

### Above-the-Fold Contract (UIUX-02)

On a 375px x 667px mobile viewport (iPhone SE) with 56px nav:

- Visible without scrolling: section tag, h1 headline, credential badges (compact inline), CTA buttons
- Below the fold (acceptable): subtitle paragraph, terminal animation
- Mobile hero order: tag -> h1 -> credentials (compact) -> CTA buttons -> subtitle -> terminal

On a 1440px desktop viewport:

- Visible without scrolling: full hero including terminal animation, CTA buttons, credential badges
- "Book a Consultation" button in nav is always visible at all viewport sizes

**Source:** RESEARCH.md Pitfall 3 (CTA not above fold). CONTEXT.md locked decisions (hero content, credentials before services).

## Motion and Animation

| Animation               | Trigger                                        | Duration | Easing   | Reduced Motion Behavior                                                         |
| ----------------------- | ---------------------------------------------- | -------- | -------- | ------------------------------------------------------------------------------- |
| Scroll reveal (fade up) | Element enters viewport (IntersectionObserver) | 600ms    | ease-out | Instant display, no animation (`opacity: 1; transform: none; transition: none`) |
| Button hover            | `:hover` pseudo-class                          | 200ms    | ease     | No change (color transition is subtle enough)                                   |
| Card hover (lift)       | `:hover` pseudo-class                          | 200ms    | ease     | No transform; color change only                                                 |
| Terminal cursor blink   | Continuous keyframe                            | 1s loop  | step-end | `animation: none` (static cursor)                                               |
| Nav background blur     | Always active (backdrop-filter)                | N/A      | N/A      | No change (not motion)                                                          |
| Mobile menu open/close  | Menu button click                              | 250ms    | ease     | Instant show/hide, no slide animation                                           |
| FAQ accordion expand    | Click on question                              | 200ms    | ease     | Instant expand, no height animation                                             |

All motion respects `@media (prefers-reduced-motion: reduce)` per CONTEXT.md locked decision.

Existing implementation: `src/scripts/scroll-reveal.ts` with CSS `.reveal` class. Do NOT replace with a framework.

**Source:** CONTEXT.md locked decisions (subtle animations, prefers-reduced-motion). RESEARCH.md Don't Hand-Roll (scroll animations).

## Interaction Contracts

### WhatsApp Floating Button (Mobile)

- Position: fixed, bottom-right corner
- Size: 56px x 56px on mobile (meets 44px minimum touch target)
- Size: 48px x 48px on desktop
- Always visible (sticky) on mobile per CONTEXT.md locked decision
- z-index: `--z-whatsapp` (70)
- Must not overlap with cookie banner or nav CTA
- Bottom offset: 24px from viewport bottom, 24px from right edge
- Accessible: `aria-label="Chat on WhatsApp"`, sr-only text

### Nav CTA (Mobile-Visible)

- On mobile (below 768px): "Book a Consultation" button visible in nav-end area, outside hamburger menu
- Compact styling: 0.75rem font, `--accdark` background, white text, 6px border-radius
- On desktop: full "Book a Consultation" link in nav-links area (existing `.nav-cta` class, text updated)
- Must not compete visually with WhatsApp float (different screen position)

### Contact Form States

| State      | Visual Treatment                                                                   |
| ---------- | ---------------------------------------------------------------------------------- |
| Default    | Input borders: `--dim`, background: `--bg2`                                        |
| Focus      | Border: `--acc`, subtle `--shadow-sm`, outline offset 2px                          |
| Error      | Border: `--destructive` (#ef4444), error text below field in `--destructive` color |
| Submitting | Button text: "Sending...", button disabled, reduced opacity (0.7)                  |
| Success    | Green check icon + success message in `--acc` color                                |
| Failure    | Error message in `--destructive` + WhatsApp alternative link                       |

### Card Hover States

| Element           | Hover Treatment                                                                                                                    |
| ----------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| Service cards     | Background shifts to `--bg3`, border becomes `rgba(52, 211, 153, 0.15)`, subtle upward transform `translateY(-2px)`, `--shadow-md` |
| Blog cards        | Same as service cards                                                                                                              |
| Credential badges | Background shifts to `--bg3`, border glow `rgba(52, 211, 153, 0.1)`                                                                |
| FAQ questions     | Text color shifts to `--white`, chevron rotates if expanded                                                                        |

All hovers use `transition: all 0.2s ease`. All transforms disabled under `prefers-reduced-motion: reduce`.

**Source:** CONTEXT.md (WhatsApp prominence, touch targets). RESEARCH.md Pattern 3 (nav CTA). UI/UX Pro Max skill (touch-target-size, hover-vs-tap, cursor-pointer).

## Accessibility Contract

| Requirement                     | Specification                                                                                        |
| ------------------------------- | ---------------------------------------------------------------------------------------------------- |
| Color contrast (body text)      | 4.5:1 minimum ratio (WCAG AA). Dark: #d4dce8 on #0b1421 = 10.5:1. Light: #2d3748 on #eef3fb = 9.8:1. |
| Color contrast (secondary text) | 4.5:1 minimum. Dark: #8494a7 on #0b1421 = 5.1:1. Light: #4a5568 on #eef3fb = 6.9:1.                  |
| Touch targets                   | 44px x 44px minimum on all interactive elements (CONTEXT.md locked)                                  |
| Focus indicators                | 2px solid `--acc` outline with 2px offset on all focusable elements                                  |
| Skip link                       | Existing `SkipLink.astro` component preserved -- links to `#main`                                    |
| ARIA labels                     | All icon-only buttons (theme toggle, hamburger menu, WhatsApp float) must have `aria-label`          |
| Keyboard navigation             | Tab order matches visual order. All interactive elements reachable via keyboard.                     |
| Reduced motion                  | `@media (prefers-reduced-motion: reduce)` disables all transforms and animations                     |
| Heading hierarchy               | One `h1` per page. Sections use `h2`. Subsections use `h3`. No skipped levels.                       |

**Source:** CONTEXT.md (touch targets, prefers-reduced-motion). UI/UX Pro Max skill (CRITICAL priority: accessibility, touch & interaction). REQUIREMENTS.md UIUX-08.
