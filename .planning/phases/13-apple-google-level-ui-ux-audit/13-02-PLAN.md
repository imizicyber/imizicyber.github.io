**TYPOGRAPHY TOKEN AUDIT — These font sizes exist outside the token scale and should be normalized:**

The token scale is: --text-xs (0.75rem/12px), --text-sm (0.84rem/13.4px), --text-base (0.938rem/15px), --text-lg (1.05rem/16.8px), --text-xl (1.25rem/20px), --text-2xl (1.5rem/24px), --text-3xl (1.875rem/30px), --text-4xl (2.25rem/36px), --text-5xl (3rem/48px).

The following orphan font-sizes are acceptable because they are between scale steps or serve specific design purposes. Do NOT change them — they were intentionally set. The goal is to verify consistency, NOT to force everything into tokens. Only change values that are clearly wrong or inconsistent with their siblings.

Specific fixes:

1. **ProcessSection.astro** — `.step p` has `font-size: 0.68rem` (10.9px), which is below the 12px minimum for readability. Change to `font-size: var(--text-xs);` (0.75rem/12px). This is the process step description text.

2. **CredentialsSection.astro** — `.cred-badge` has `font-size: 0.56rem` (9px), which is acceptable for badge labels (similar to `.training-label` at 0.56rem in ServicesSection). Leave as-is. BUT `.cred-info span` at `font-size: 0.66rem` (10.6px) is too small for the credential organization name. Change to `font-size: var(--text-xs);` (0.75rem/12px).

3. **global.css** — `.breadcrumb` has `font-size: 0.64rem` (10.2px). This is a navigation breadcrumb and should be at least readable. Change to `font-size: var(--text-xs);` (0.75rem/12px).

4. **global.css** — `.blog-date` has `font-size: 0.58rem` (9.3px). Change to `font-size: 0.65rem;` — this is a date stamp so can be smaller than body but 9.3px is too small.

5. **global.css** — `.footer-left` has `font-size: 0.68rem` (10.9px). Change to `font-size: var(--text-xs);` (0.75rem/12px).

**MOBILE TOUCH TARGET AUDIT:**

7. **Nav.astro** — The `.menu-btn` has `padding: 10px` in global.css. On mobile, verify this produces at least 44x44px. The hamburger spans are 20px wide with 2px height each and 4px margin. Total button area: 20px + 20px padding = 40px wide. The height: 3 _ 2px + 2 _ 4px = 14px + 20px padding = 34px. This is below 44px. Add `min-height: 44px; min-width: 44px;` to the `.menu-btn` rule in global.css.

8. **global.css** — `.theme-toggle` has `padding: 5px 7px` and SVG is 15x15px. Total: 15+14=29px wide, 15+10=25px tall. This is below 44px minimum. Add `min-height: 44px; min-width: 44px; justify-content: center;` to the `.theme-toggle` rule in global.css.

9. **global.css** — `.faq summary` has `padding: 12px 16px` and font-size 0.82rem. The line-height and padding should produce a tall enough target. On 480px, padding drops to `10px 12px`. With ~13px font and 10px padding top/bottom = 33px total, which is below 44px. Change the 480px override for `.faq summary` from `padding: 10px 12px` to `padding: 12px 14px` (maintaining the 44px minimum).

10. **global.css** — `.footer-social a` has no padding or size. The SVGs are 18x18px. Add `min-height: 44px; min-width: 44px; display: inline-flex; align-items: center; justify-content: center;` to `.footer-social a`.

**SPACING TOKEN CONSISTENCY:**

11. **global.css** — `.sec` padding is `60px 0`. Convert to `padding: var(--space-16) 0;` (64px). The 4px difference is negligible and gains token consistency.

12. **global.css** — `.hero` padding is `132px 0 48px`. The 48px bottom is close to `var(--space-12)` (48px) — already matches. The 132px is nav-height-adjusted and intentional, leave as-is.

13. **global.css** — Container `.w` padding `0 24px` is close to `var(--space-6)` (24px) — already matches. No change needed.

**MOBILE TEXT SIZE FLOOR:**

14. **global.css** — In the 480px media query, add a minimum font-size guard to ensure body text never goes below 14px (0.875rem). The current body font-size is 15px which is fine, but some scoped styles on small screens go below. Add to the 480px query:

```css

  font-size: max(0.84rem, 0.875rem);

Actually, 13.4px is very close to 14px and the existing Playwright responsive tests may assert current sizes. To avoid breaking tests, leave the 480px article-body sizes as-is and instead just ensure no font-size in the codebase is below 0.64rem (10.2px) for labels or below 0.72rem (11.5px) for readable body text. The fixes above (items 1-6) already address the worst offenders.
