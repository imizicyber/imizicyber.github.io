   - Update button structure:
     ```html
     <button type="submit" class="submit-btn" aria-describedby="form-status">
       <span class="btn-text">Send enquiry</span>
       <span class="spinner" aria-hidden="true"></span>
     </button>
     ```

3. Update the form submission script to use ARIA states per CONTEXT.md locked decision:

   On submit start:
   - `form.setAttribute('aria-busy', 'true')`
   - `btn.disabled = true`
   - `btn.querySelector('.btn-text').textContent = 'Sending...'`
   - `btn.querySelector('.spinner').classList.add('visible')`
   - Hide any previous success/error: set `hidden` attribute on both message divs

   On success:
   - `form.setAttribute('aria-busy', 'false')`
   - Remove `hidden` from success div (screen reader will announce via aria-live)
   - `form.reset()`
   - `btn.querySelector('.btn-text').textContent = 'Sent!'`
   - `btn.querySelector('.spinner').classList.remove('visible')`
   - After 3 seconds, re-enable button: `btn.disabled = false`, text back to 'Send enquiry'

   On error:
   - `form.setAttribute('aria-busy', 'false')`
   - Remove `hidden` from error div (screen reader will announce via aria-live)
   - `btn.querySelector('.btn-text').textContent = 'Send enquiry'`
   - `btn.querySelector('.spinner').classList.remove('visible')`
   - `btn.disabled = false`

   IMPORTANT: Replace all `style.display = 'block'` / `style.display = 'none'` with `hidden` attribute toggling. The `style="display:..."` pattern is flagged in RESEARCH.md anti-patterns and was also an issue for CSP in Phase 2.
