The WhatsAppFloat component currently accepts a `message` prop (display label) but always links to `CONTACT.whatsappUrl` (generic greeting). The `whatsappMessage` prop is passed from service pages through BaseLayout but the float component ignores it for the URL.

Update `WhatsAppFloat.astro`:

1. Add a new prop `prefilledText` (optional string) to the Props interface. This is the WhatsApp pre-filled message text (NOT the button label).

2. Compute the href dynamically:

   ```typescript
   const phone = '250793146617';
   const defaultText = "Hi imizicyber, I'd like to discuss security services";
   const text = prefilledText || defaultText;
   const whatsappHref = `https://wa.me/${phone}?text=${encodeURIComponent(text)}`;
   ```

3. Use `whatsappHref` instead of `CONTACT.whatsappUrl` in the anchor href. Remove the CONTACT import since it is no longer needed.

4. Keep the existing trackEvent script unchanged — it already tracks whatsapp_click with page path.

Then update `src/layouts/BaseLayout.astro` to pass whatsappMessage through to WhatsAppFloat as the `prefilledText` prop:
Change: `<WhatsAppFloat message={whatsappMessage} />`
To: `<WhatsAppFloat prefilledText={whatsappMessage} />`

The `message` prop (display label) can stay at its default "Chat with us" — no need to change it.

**Track inline WhatsApp CTA links on service pages:**
The service pages (penetration-testing, security-assessments, custom-tooling) have inline `<a href="https://wa.me/...">` links in their CTA sections that are NOT tracked. Add a small inline script at the bottom of each service page that finds these inline WhatsApp links and adds click tracking:

For each of the three flagship service pages, add a `<script>` block:

```astro
<script>
  import { trackEvent } from '@/scripts/analytics';
  document.querySelectorAll('a[href*="wa.me"]').forEach((link) => {
    if (!link.closest('.wa-float')) {
      link.addEventListener('click', () => {
        trackEvent('whatsapp_click', {
          page: window.location.pathname,
          location: 'cta_section',
        });
      });
    }
  });
</script>
```

This targets inline WhatsApp links while excluding the floating button (which has its own tracker).
