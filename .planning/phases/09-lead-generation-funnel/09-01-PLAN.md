  <files>package.json, src/scripts/quiz/pdf.ts, src/scripts/resources.ts</files>
  <action>
1. Run `npm install jspdf` to add jsPDF as a local dependency.

2. In `src/scripts/quiz/pdf.ts`:
   - Remove the `loadJsPDF()` function that dynamically creates script tags from CDN URLs.
   - Remove the `window.jspdf` global declaration and all references to `window.jspdf`.
   - Add `import { jsPDF } from 'jspdf';` at the top.
   - Remove the custom `JsPDFInstance` interface — use the real jsPDF type from the package instead. The `jsPDF` class from the package provides all the methods currently in `JsPDFInstance` (setFillColor, rect, text, splitTextToSize, etc.).
   - Update `generatePDF()` to create `const doc = new jsPDF()` directly instead of `new window.jspdf.jsPDF()`.
   - Export a synchronous `createJsPDF()` function that returns `new jsPDF()` for use by resources.ts (replacing the async loadJsPDF pattern).
   - Keep `esc()` and `fallbackHTML()` exports unchanged.

3. In `src/scripts/resources.ts`:
   - Replace `import { loadJsPDF, esc } from './quiz/pdf'` with `import { createJsPDF, esc } from './quiz/pdf'` and `import { jsPDF } from 'jspdf'`.
   - Remove the `JsPDFInstance` type import.
   - In `generateResourcePDF()`, replace `const jspdfMod = window.jspdf; const doc = new jspdfMod.jsPDF()` with `const doc = new jsPDF()`.
   - In `submitResource()`, replace the `loadJsPDF().then(...)` pattern with a synchronous call: just call `generateResourcePDF(resource, name, org)` directly inside a try/catch, falling back to `resourceFallbackHTML()` on error.

4. Run `npm run typecheck` and `npm run build` to verify everything compiles. The jsPDF UMD module is ~280KB but Astro's build will tree-shake and bundle it — no runtime CDN requests needed.

Note: The `setFont(undefined, 'bold')` calls use jsPDF's real API where undefined means "keep current font family". This works with the npm package the same as the CDN version.
