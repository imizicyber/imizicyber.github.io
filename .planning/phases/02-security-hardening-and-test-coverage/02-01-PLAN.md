  <action>
Add a custom ESLint rule to `eslint.config.mjs` that bans innerHTML usage across the entire codebase. Add a `rules` block to the existing config:

```javascript
{
  rules: {
    'no-restricted-properties': [
      'error',
      {
        object: 'document',
        property: 'innerHTML',
        message: 'Use DOM API (createElement + textContent) instead of innerHTML for XSS safety.',
      },
    ],
    'no-restricted-syntax': [
      'error',
      {
        selector: "AssignmentExpression[left.property.name='innerHTML']",
        message: 'innerHTML is banned. Use DOM API (createElement + textContent + appendChild) instead.',
      },
      {
        selector: "MemberExpression[property.name='innerHTML']",
        message: 'innerHTML is banned. Use DOM API (createElement + textContent + appendChild) instead.',
      },
    ],
  },
}
```

This catches both `element.innerHTML = ...` and `document.innerHTML` patterns. The `no-restricted-syntax` with AST selector is the most reliable approach for catching all innerHTML usage patterns.

Run `npm run lint` to verify the rule works and no violations exist.
