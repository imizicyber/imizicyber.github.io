**Step 1: Fix the font preload inline onload handler in BaseLayout.astro.**

The `onload="this.onload=null;this.rel='stylesheet'"` on line 108 is an inline event handler. CSP Level 2 does NOT hash inline event handlers. Replace with a blocking stylesheet load:

Change lines 104-115 (the preload+noscript block) to a simple blocking stylesheet:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700;800&family=JetBrains+Mono:wght@400;500&display=swap"
/>
```

Remove the `<noscript>` fallback block (no longer needed with blocking load). This is a minor performance tradeoff (blocking vs async font load) but necessary for CSP compliance. Performance optimization deferred to Phase 4.

**Step 2: Enable Astro experimental CSP in astro.config.mjs.**

Add `experimental` and `security` blocks to `astro.config.mjs`:

```javascript
experimental: {
  csp: true,
},
security: {
  csp: {
    algorithm: 'SHA-256',
    directives: [
      "default-src 'self'",
      "font-src https://fonts.gstatic.com",
      "img-src 'self' https: data:",
      "connect-src 'self' https://formspree.io https://www.google-analytics.com",
      "frame-ancestors 'none'",
    ],
    scriptDirective: {
      resources: ["'self'", "https://www.googletagmanager.com"],
    },
    styleDirective: {
      resources: ["'self'", "https://fonts.googleapis.com"],
    },
  },
},
```

**Step 3: Remove manual CSP meta tag from BaseLayout.astro.**

Delete the entire CSP construction block (lines 57-66):

**Step 1: Create `.gitleaks.toml` allowlist.**

Create `.gitleaks.toml` in the project root:

```toml
title = "Imizi Cyber Gitleaks Config"

[allowlist]
  description = "Intentionally public identifiers"
  regexTarget = "match"
  regexes = [
    '''G-R7TC88KH9N''',
    '''https://formspree\.io/f/''',
  ]
  paths = [
    '''\.planning/.*''',
    '''\.gitleaks\.toml''',
  ]
```

The GA ID `G-R7TC88KH9N` and Formspree endpoint are intentionally public (embedded in client-side HTML). Planning docs may reference these IDs in discussion.

**Step 2: Verify gitleaks is installed.**

Run `which gitleaks`. If not installed, run `brew install gitleaks`. If brew is not available (CI environment), document the requirement.

**Step 3: Run full git history scan.**

Run `gitleaks detect --source . --verbose --config .gitleaks.toml`. This scans ALL commits in the repo history.

If findings are reported:

- For each finding, determine if it is a real secret or a false positive
- Real secrets: Document in the summary what was found and that the user must rotate the credential. Do NOT attempt to rewrite git history automatically.
- False positives: Add to `.gitleaks.toml` allowlist and re-run

The scan must exit 0 (no findings after allowlist) before proceeding.

**Step 4: Add gitleaks to pre-commit hook.**

Update `.husky/pre-commit` to add gitleaks as the final step:

```bash
npm run typecheck
npm run test -- --run
npx lint-staged
gitleaks protect --staged --redact --config .gitleaks.toml
```

The `--staged` flag scans only staged changes. The `--redact` flag masks any found secrets in output. The `--config` flag points to the allowlist.

**Step 5: Verify the hook works.**

Stage a test file, verify the pre-commit hook runs all 4 steps including gitleaks. Then unstage the test file.
