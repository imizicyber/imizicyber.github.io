```json
      }

    "upload": {
      "target": "temporary-public-storage"
    }
  }

```

Key decisions in this config:

- `staticDistDir: ./dist` — LHCI serves static files directly (no separate webServer needed)
- 5 representative URLs — homepage, about, blog index, a service page, and the quiz tool
- `numberOfRuns: 3` — median aggregation for stable scores
- `total-blocking-time` at 200ms — maps to CONTEXT.md "FID < 100ms" requirement (TBT is the Lighthouse lab proxy for responsiveness since FID/INP require real user interaction)
- `categories:performance` as `warn` not `error` — CI performance scores can vary; hard metrics (LCP, CLS, TBT) are the errors
- `categories:accessibility` as `error` with minScore 0.9 — strict accessibility enforcement
- `temporary-public-storage` — works without GitHub App token setup

  <action>
Create `.github/workflows/lighthouse.yml` with the following exact content:

```yaml
name: Lighthouse CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lighthouse:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version-file: .nvmrc
          cache: npm

      - run: npm ci

      - name: Build site
        run: npm run build

      - name: Run Lighthouse CI
        run: npx @lhci/cli autorun
        env:
          LHCI_GITHUB_APP_TOKEN: ${{ secrets.LHCI_GITHUB_APP_TOKEN }}
```

Key decisions:

- Follows same trigger pattern as `test-e2e.yml` (push + PR to main)
- Uses `actions/setup-node@v4` with `node-version-file: .nvmrc` and `cache: npm` (same as existing workflows)
- Runs `npm run build` (which includes CSP injection) before Lighthouse
- Uses `npx @lhci/cli autorun` instead of globally installing — ensures version from devDependencies is used
- `LHCI_GITHUB_APP_TOKEN` is optional — if the secret is not configured, LHCI still runs and reports results, it just does not post status checks to PRs. The `temporary-public-storage` upload target works without any token.
- Separate from E2E workflow — Lighthouse runs independently and does not count against Playwright suite runtime (per CONTEXT.md: keep suite under 2 minutes)
